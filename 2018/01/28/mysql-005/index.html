mysql 学习(五) mysql 数据类型

发表于 2018-01-28 | 分类于 数据库

数据类型

  • 选择合适的数据类型来存储数据;

  • 数据优化, 选择最合适的数据类型;

整型

有符号位: 指的是数字最高为用 0 或 1 表示正负数;
无符号位: 没有负数, 用完都是整数;

数据类型 存储范围 字节
TINYINT 有符号值: -2的7次方到2的7次方减1; 无符号值: 0 到 2的8次方减1 1
SMALLINT 有符号值: -2的15次方到2的15次方减1; 无符号值: 0 到 2的16次方减1 2
MEDIUMINT 有符号值: -2的23次方到2的23次方减1; 无符号值: 0 到 2的24次方减1 3
INT 有符号值: -2的31次方到2的31次方减1; 无符号值: 0 到 2的32次方减1 4
BIGINT 有符号值: -2的63次方到2的63次方减1; 无符号值: 0 到 2的64次方减1 8

浮点型

  • FLOAT[(M, D)] 或者 DOUBLE[(M, D)]

    M 是数字总位数, D 是小数点后面的位数; 一定是 M >= D;

  1. FLOAT型:
    保留带小数点的后七位;

  2. DOUBLE型:
    是 FLOAT 型的10倍;(一般用不到);

日期时间型

一般采用数字表示(跨时区);

  2. TIME(3):

  4. DATETIME(8):
    5: TIMESTAMP(4):

字符型

  1. CHAR(M): M 个字节 0 <= M <= 255, 定长类型, 没有达到自定的长度用 空格补齐;
  2. VARCHAR(M): 0 <= M <= 65535, 不定长;
  3. TINYTEXT: L + 1 个字节; L < 2 的 8 次方;
  4. TEXT: L + 2 个字节; L < 2 的 16 次方;
  5. MEDIUMTEXT: L + 3 个字节; L < 2 的 24 次方;
  6. LONGTEXT: L + 4 个字节; L < 2 的 32 次方;
  7. ENUM(‘value1’, ‘value2’,…): 1 或 2 字节, 取决于枚举值的个数(最多65535个值);
  8. SET(‘value1’, ‘value2’,…): 1, 2, 3, 4 或 8 个字节, 取决去 set 的成员数目(最多64个成员);
