攻击类型

    发出请求时, XSS 代码出现在 URL 中, 作为输入提交到服务器端,
    服务器端解析后响应, XSS代码随响应内容一起传回给浏览器,
    最后浏览器解析执行 XSS 代码.

    明文, 在 URL 中

    html 中有部分是后端解析返回的模板渲染的

  2. 存储型;

  1. 编码
    • 对用户输入进行编码: HTML Entity

mysql 学习(五) mysql 数据类型

发表于 2018-01-28 | 分类于 数据库

数据类型

  • 选择合适的数据类型来存储数据;

  • 数据优化, 选择最合适的数据类型;

整型

有符号位: 指的是数字最高为用 0 或 1 表示正负数;
无符号位: 没有负数, 用完都是整数;

数据类型 存储范围 字节
TINYINT 有符号值: -2的7次方到2的7次方减1; 无符号值: 0 到 2的8次方减1 1
SMALLINT 有符号值: -2的15次方到2的15次方减1; 无符号值: 0 到 2的16次方减1 2
MEDIUMINT 有符号值: -2的23次方到2的23次方减1; 无符号值: 0 到 2的24次方减1 3
INT 有符号值: -2的31次方到2的31次方减1; 无符号值: 0 到 2的32次方减1 4
BIGINT 有符号值: -2的63次方到2的63次方减1; 无符号值: 0 到 2的64次方减1 8

浮点型

  • FLOAT[(M, D)] 或者 DOUBLE[(M, D)]

    M 是数字总位数, D 是小数点后面的位数; 一定是 M >= D;

  1. FLOAT型:
    保留带小数点的后七位;

  2. DOUBLE型:
    是 FLOAT 型的10倍;(一般用不到);

日期时间型

一般采用数字表示(跨时区);

  1. YEAR(1):
  2. TIME(3):
  3. DATE(3):
  4. DATETIME(8):
    5: TIMESTAMP(4):

字符型

  1. CHAR(M): M 个字节 0 <= M <= 255, 定长类型, 没有达到自定的长度用 空格补齐;
  2. VARCHAR(M): 0 <= M <= 65535, 不定长;
  3. TINYTEXT: L + 1 个字节; L < 2 的 8 次方;
  4. TEXT: L + 2 个字节; L < 2 的 16 次方;
  5. MEDIUMTEXT: L + 3 个字节; L < 2 的 24 次方;
  6. LONGTEXT: L + 4 个字节; L < 2 的 32 次方;
  7. ENUM(‘value1’, ‘value2’,…): 1 或 2 字节, 取决于枚举值的个数(最多65535个值);
  8. SET(‘value1’, ‘value2’,…): 1, 2, 3, 4 或 8 个字节, 取决去 set 的成员数目(最多64个成员);

盒模型

组成

  1. content: 内容区块;
  2. padding: 内边距;
  3. border: 边框;
  4. margin: 外边距;

标准盒模型和怪异盒模型

盒模型分为两种:

  1. W3C 标准盒模型;
  2. IE 标准盒模型;

怪异模式

部分浏览器在支持W3C标准的同时还保留了原来的解析模式;

主要表现在 IE 内核的浏览器, 当不对 Doctype 进行定义的时,触发 怪异模式.

计算

标准盒模型

> 块的总宽 = width + margin(左右) + padding(左右) + border(左右);

怪异和模型

> width = content的宽 + padding(左右) + border(左右);

> 块的总宽 = width + margin(左右);

box-sizing 属性

  • 语法

  • 当设置 border-box 时, 采用怪异模式解析计算;
  • 当设置 content-box 时, 采用 w3c 标准解析计算;

    建议初始化是进行 box-sizing: border-box; 设置.

简单的命令

  1. 安装软件：brew install 软件名;
  2. 搜索软件：brew search 软件名;
  3. 卸载软件：brew uninstall 软件名;

  4. 更新所有软件：brew update;

  8. 查看那些已安装的程序需要更新： brew outdated;
